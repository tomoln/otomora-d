# 🎨 Electron + WebGL でつくる「セリフ音声ジェネレーティブ・ビジュアライザー」  
## — 分かりやすく整理した要件とアート方向性 —

## 🧩 **1. 基本構成**
- UI は **Electron** 上で動作する。
- 入力として  
  - **セリフ入りの音声ファイル（.wav）**  
  - **対応する JSON**（単語・モーラのタイムスタンプ、rms / f0 / spectral_centroid / zcr を含む）  
  を読み込む。

## 🕒 **2. タイムスタンプ同期**
- JSON に含まれる  
  - **単語の開始/終了時刻**  
  - **モーラの開始/終了時刻**  
  を参照し、WebGL 上のアニメーションを **音声と完全同期** させる。

## 🎚️ **3. 音響パラメータをアートに反映**
JSON に含まれる以下の値を、リアルタイムのエフェクトとして使用する：

| パラメータ | 意味 | アートでの使い方例 |
|-----------|------|---------------------|
| f0 | ピッチ | 色相、形の高さ、揺れの速さ |
| rms | 音量 | 大きさ、明るさ |
| spectral_centroid | 明るさ | 色温度、シャープさ、グロー量 |
| zcr | ノイズ感 | グリッチ |

---

# 🌈 **4. アート方向性：Analogous / Complementary を使ったジェネレーティブ表現**

## 🎨 **Complementary（補色）を使う理由**
- 色相環で正反対の 2 色を使う（例：青 × オレンジ）
- コントラストが強く、音声の変化を視覚的に強調できる
- f0 の高低や、単語の切り替わりなど「対立する情報」を表現しやすい

### 🔧 具体的な使い方例
- **f0 が高い → オレンジ系にシフト**  
- **f0 が低い → 青系にシフト**  
- 単語の切り替わりで背景を補色側にパッと反転させる
