
001 - オーディオ取り込む
「ファイルを開く」のは Node.js
「音声データとして解釈する」のは Web Audio API

002 - 取り込んだオーディオのスライス
この機能は Web Audio API の役割です。
オーディオとjsonのタイムスタンプの位置対応は機能にないので、
JavaScript のロジックを自分で書く必要がある。
オーディオのスライスをjsonに従って、位置を決める
元の音声全体は fullBuf に入っていて、
sliceBuffer(audioCtx, fullBuf, m.start, m.end) が 1 モーラ分だけ切り出す。
その返り値を moras.map(...) で順番に集めたものが slicedBufs。
つまり、slicedBufs自体にスライスされたオーディオファイル、
厳密には分割済みの AudioBuffer 群が入っている。

003 - オーディオファイルの再生
Web Audio API が担当

004 - 再生中のBPMを変える
タイムストレッチはsoundtouchjsの役割です。
タイムストレッチには soundtouchjs`（バージョン `^0.2.3`）を使用する
BPMが早いほど、オーディオもタイムストレッチで短くされる(次の音と重ならないため)
これは分割されたslicedBufsの音にそれぞれ適応される。
また、次のオーディオと音が重なる場合も想定して、前の音を、次の音が再生される直前で、短めのフェードアウトする
タイムストレッチは、ピッチは変えず、速度だけ変わること


005 - エフェクトを適用させる
タイムストレッチが必要なら、soundtouchjsに渡して Web Audio APIに戻すし、タイムストレッチがないなら、Web Audio API内で完結させる

ピッチドロップ
ピッチが段々落ちていって、最終的に0になる
Web Audio APIで実装可能

フリーズ
現在の音がグラニュラー的にフリーズしてなり続ける
Web Audio APIで実装可能

グラニュラーフリーズの仕組み：
フリーズ位置の前後 100〜200ms を「グレイン」として切り出す
  ↓
複数の AudioBufferSourceNode でグレインを少しずつズラして重ねて再生
  ↓
各グレインに GainNode でハン窓（フェードイン→フェードアウト）をかける
  ↓
グレインが途切れ目なく重なり合い、連続した音に聞こえる

スタッター
16,32,64,128で分割可能
Web Audio APIで実装可能
1小節 = 4拍 = 4 * beatSec 秒
  ↓ 128分割
grainSec = (4 * beatSec) / 128

slicedBufs[n] を grainSec の長さで繰り返しスケジュール
