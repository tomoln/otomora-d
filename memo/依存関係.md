# 依存関係

## 全体の流れ（迷ったときの起点）

```
main/main.js
  ├── main/link/linkBridge.js   ← Ableton Link（BPM）
  └── main/midi/midiMain.js     ← MIDIコントローラー
       ↕ IPC（ipcMain / ipcRenderer）
renderer/index.js
  ├── store/AppStore.js
  ├── audio/ ...
  ├── effects/ ...
  ├── midi/ ...
  └── ui/ ...
```

メインプロセスとレンダラープロセスは直接 import できない。
すべての情報は **IPC（`ipcMain.on` / `ipcRenderer.on`）** を経由してやり取りする。

---

## ファイルごとの依存関係

### main/ （メインプロセス）

#### `main/main.js`
| 依存先 | 種類 | 何のため |
|---|---|---|
| `main/link/linkBridge.js` | ローカル | Link の起動・IPC登録 |
| `main/midi/midiMain.js` | ローカル | MIDI の起動・IPC登録 |
| `electron` | npm | BrowserWindow, ipcMain |
| `path`, `fs` | Node.js 標準 | ファイルパス解決、assets一覧取得 |

#### `main/link/linkBridge.js`
| 依存先 | 種類 | 何のため |
|---|---|---|
| `abletonlink`（または類似）| npm ネイティブ | BPM・ビート位置の取得 |
| `electron` | npm | `ipcMain` でレンダラーへ送信 |

> **迷ったとき:** BPMがレンダラーに届かない → ここの `ipcMain.handle` か、受け取り側 `BpmSync.js` の `ipcRenderer.on` を確認。

#### `main/midi/midiMain.js`
| 依存先 | 種類 | 何のため |
|---|---|---|
| `node-midi` | npm | MIDIデバイス列挙・メッセージ受信 |
| `electron` | npm | `ipcMain` でレンダラーへ転送 |

### audio/

#### `audio/AudioEngine.js`
**他のファイルへの依存：なし**（最も根本のファイル）

| 持つもの | 内容 |
|---|---|
| `AudioContext` | Web Audio API のコンテキスト（全音声処理の基盤） |
| `masterGainNode` | 出力ゲート |

> **迷ったとき:** 音が一切出ない → まずここの `AudioContext` が `running` 状態か確認。

---

#### `audio/SliceManager.js`
| 依存先 | 種類 | 何のため |
|---|---|---|
| `audio/AudioEngine.js` | ローカル | `decodeAudioData` でWAVをバッファに変換 |
| `electron` | npm | `ipcRenderer.invoke` でメインに assets のパスを問い合わせ |

> **持つもの:** `slices[]` — `{ buffer, word, mora, startTime, rms, f0, spectral_centroid, zcr }` の配列

> **迷ったとき:** スライスの数がおかしい、単語がずれる → JSONのタイムスタンプとここの切り出しロジックを確認。

---

#### `audio/TimeStretcher.js`
| 依存先 | 種類 | 何のため |
|---|---|---|
| `audio/AudioEngine.js` | ローカル | 処理結果を AudioContext に流す |
| `soundtouchjs ^0.2.3` | npm | ピッチを保ったまま再生速度を変える |

> **迷ったとき:** BPMを変えても速度が変わらない / 音が歪む → ここの `tempo` 設定値を確認。

---

#### `audio/FadeManager.js`
| 依存先 | 種類 | 何のため |
|---|---|---|
| `audio/AudioEngine.js` | ローカル | `GainNode` の `linearRampToValueAtTime` でフェード |

> **迷ったとき:** 音が重なって汚い → ここのフェードタイミング計算を確認。

---

#### `audio/BpmSync.js`
| 依存先 | 種類 | 何のため |
|---|---|---|
| `audio/AudioEngine.js` | ローカル | `AudioContext.currentTime` でスケジューリング |
| `electron` | npm | `ipcRenderer.on('link-beat')` でBPM受信 |
| `store/AppStore.js` | ローカル | 現在のBPM・x2状態を読み書き |

> **持つもの:** `onBeat(callback)` — グリッドの各16分音符のタイミングでコールバックを呼ぶ

> **迷ったとき:** グリッドとAbletonのタイミングがずれる → IPCで届くビート位置と `AudioContext.currentTime` のオフセット補正をここで確認。

## 依存の方向まとめ（上が下に依存する）

```
renderer/index.js
│
├─ ui/*.js
│   └─ audio/GridPlayer.js ←─────────┐
│       ├─ audio/SliceManager.js      │
│       ├─ audio/TimeStretcher.js     │
│       ├─ audio/BpmSync.js ←── IPC ─┼── main/link/linkBridge.js
│       ├─ audio/FadeManager.js       │
│       └─ audio/AudioEngine.js       │
│                                     │
├─ effects/EffectChain.js             │
│   ├─ effects/*.js                   │
│   └─ audio/AudioEngine.js           │
│                                     │
├─ midi/MidiMapping.js                │
│   ├─ midi/MidiController.js ← IPC ─┼── main/midi/midiMain.js
│   └─ store/AppStore.js              │
│                                     │
└─ store/AppStore.js（依存なし）      │
   audio/AudioEngine.js（依存なし）   │
```

## バグを追うときの手順

1. **音が出ない** → `AudioEngine.js` の `AudioContext.state` を確認 → `GridPlayer.js` の再生呼び出しを確認
2. **タイミングがずれる** → `BpmSync.js` の IPC 受信 → `AudioContext.currentTime` のオフセット補正
3. **スライスがおかしい** → `SliceManager.js` の JSON パースと切り出し範囲
4. **エフェクトが効かない** → `EffectChain.js` のノード接続順
5. **MIDIが反応しない** → `main/midi/midiMain.js` のデバイスオープン → `MidiController.js` の IPC 受信
6. **UIが更新されない** → `AppStore.js` の状態変化が `GridPlayer` や `ui/` に伝わっているか
