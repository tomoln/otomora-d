# 仕様: MIDI ボタンマッピング テスト

## 概要

接続されたMIDIコントローラーのボタンを、4つのUIボタンに自由に割り当てるテスト。
割り当て後は、MIDIボタンを押すとUIボタンが発火する。

## 実行

```bash
npm run test:midi_send
```

## UI構成

- **デバイス表示**: 接続中のMIDI入力デバイス名をウィンドウ上部に表示
- **ボタンスロット × 4**: 2列グリッドで配置
  - **UIボタン**: クリックで手動発火
  - **「割り当て」ボタン**: MIDI Learnモードの開始/キャンセル
  - **「✕」ボタン**: 割り当て解除
  - **マッピングラベル**: 現在の割り当て内容を表示（例: `Note 36 (ch1)`）
- **ステータス表示**: 操作の状態をウィンドウ下部に表示

## MIDI Learn フロー

1. 「割り当て」ボタンをクリック → スロットが橙色に点滅、Learnモード開始
2. MIDIコントローラーのボタンを押す → マッピング登録、UIボタンが1回発火
3. 以降、そのMIDIボタンを押すたびにUIボタンが発火

- 同時に学習できるスロットは1つのみ（別スロットの「割り当て」を押すと切り替わる）
- Learnモード中に同じ「割り当て」ボタンを再クリック、または `Esc` キーでキャンセル

## 対応MIDIメッセージ

| 種類 | 条件 | ラベル例 |
|------|------|---------|
| Note On (0x9n) | velocity > 0 | `Note 36 (ch1)` |
| Control Change (0xBn) | value > 0 | `CC 74 (ch1)` |
| Program Change (0xCn) | — | `Program 5 (ch1)` |

- Note Off および CC value=0 は無視（一部コントローラーのリリース信号対策）
- 1つのMIDIメッセージを複数スロットに同時割り当て可能

## マッピングの識別キー

`type:channel:number` の文字列で一意に識別する。
チャンネルは 0-indexed で保持し、表示時に +1 する。

## デバイスの抜き差し対応

`midiAccess.onstatechange` でデバイス変更を検知し、全入力ポートを再接続する。
マッピングはメモリ上に保持（アプリ終了で消える）。

## ウィンドウサイズ

`500 × 260` px（他テストと共通の main.js 設定）
